# Product Brief: AI EDA 原理图生成器

---

## 执行摘要

**AI EDA 原理图生成器**旨在消除硬件设计的专业门槛，让没有电子工程背景的普通人也能快速、准确地完成原理图设计并投入 PCB 打板。

当前，想要动手做硬件的创客和产品人往往在原理图这一关被阻断——他们需要翻阅大量开源项目进行对照参考、依靠经验判断抄什么改什么，这个过程既耗时又难以做出正确决策。AI EDA 原理图生成器通过自然语言输入驱动 EDA 工具自动生成原理图，将这一过程从"专家才能做"变为"普通人学一学就能做"。

当更多人能够顺畅地完成原理图 → PCB 打板的完整链路，硬件产品的迭代速度将大幅提升，真正实现**人人都是创客**的愿景。

---

## 核心愿景

### 问题陈述

- **技能壁垒**：原理图绘制需要掌握元器件知识、电气规则、EDA 工具操作，学习曲线陡峭
- **参考资料分散**：找现成参考需要在 GitHub、立创开源广场、各硬件论坛中大量对比，效率极低
- **决策困难**：普通人面对多种方案不知道如何取舍，容易卡在"选哪个芯片/怎么连线"上
- **现有工具门槛高**：KiCad、Altium 等专业 EDA 工具对新手不友好

### 现有方案的不足

| 现有方案 | 不足 |
|----------|------|
| 直接学 EDA 工具 | 学习周期长，劝退大多数普通人 |
| 找开源项目参考 | 要对比多个项目，耗时、难决策 |
| 请人代画 | 成本高，沟通成本大，迭代慢 |
| AI 生成代码式原理图 | 输出格式不能直接用，脱离 EDA 生态 |

### 解决方案愿景

用户用自然语言描述电路需求（如"我要一个 5V 转 3.3V 的供电模块"），AI 理解意图、选择合适方案、通过 EDA 插件 API 直接生成可编辑的原理图文件。用户在熟悉的 EDA 环境中做最终确认和调整，门槛降至最低。

### 核心差异化

- **不脱离 EDA 生态**：直接在 EDA 工具内输出可编辑原理图，而非示意图
- **面向普通人而非专家**：目标是降低门槛，不是提升专家效率
- **AI 做决策，人做确认**：把最难的"选什么、怎么连"交给 AI
- **国内供应链闭环**：设计 → 选型 → 打板全链路贯通

## EDA 技术选型分析：立创 EDA 专业版

### 选型背景

基于概念验证（POC）阶段的目标，综合评估了主流 EDA 工具后，**推荐优先选用立创 EDA 专业版**作为首阶段实现平台。

### 立创 EDA 专业版核心能力

| 能力维度 | 详情 |
|----------|------|
| **插件扩展 SDK** | 官方支持 JavaScript/TypeScript 插件开发，可操作原理图元素 |
| **组件库** | 直连立创商城，超过 2000 万种元器件，含封装、3D 模型、价格信息 |
| **BOM 联动** | 设计完成后可直接一键导出 BOM 并跳转立创商城下单 |
| **PCB 打板联动** | 与嘉立创（JLCPCB）深度集成，原理图 → PCB → 下单全链路 |
| **用户基础** | 国内最大的硬件设计社区，目标用户群高度重合 |
| **开源广场** | 海量参考设计可作为 AI 训练/检索的知识库 |

### 可行性评估

**✅ 高可行性的部分：**
- 插件 SDK 支持通过 API 创建原理图符号、放置元器件、绘制导线
- 可访问立创商城元器件数据库进行智能选型（型号、规格、库存、价格）
- 国内用户无访问障碍，社区生态完整
- 与嘉立创打板流程天然集成，用户路径最短

**⚠️ 需要验证的部分：**
- 插件 SDK 对自动布局（Auto Placement）的支持程度
- 复杂连线逻辑（总线、多页面网络）的 API 完整性
- 插件内调用外部 LLM API 的网络策略（CORS/代理问题）
- SDK 版本稳定性与官方维护节奏

**❌ 已知限制：**
- 插件运行在渲染进程中，直接调用重型本地模型受限
- 复杂电路的自动美观布局需单独实现排版算法

### 推荐技术架构（立创 EDA 方案）

```
┌─────────────────────────────────────────┐
│           用户（立创 EDA 界面）            │
│  输入自然语言需求："ESP32 最小系统电路"    │
└──────────────┬──────────────────────────┘
               │ 插件面板触发
               ▼
┌─────────────────────────────────────────┐
│        立创 EDA 插件（JS/TS）            │
│  - 提供对话式 UI 面板                    │
│  - 调用后端 AI 服务                      │
│  - 接收结构化电路描述                    │
│  - 调用 SDK API 执行绘图动作             │
└──────────────┬──────────────────────────┘
               │ HTTP / WebSocket
               ▼
┌─────────────────────────────────────────┐
│         后端 AI 服务（本地或云端）        │
│  - LLM 解析自然语言意图                  │
│  - 匹配电路模板库                        │
│  - 查询立创商城元器件 API                │
│  - 输出结构化 JSON 电路描述              │
└─────────────────────────────────────────┘
               │ JSON 电路描述
               ▼
┌─────────────────────────────────────────┐
│      立创 EDA SDK 执行层                 │
│  placeComponent() / drawWire()          │
│  setNetLabel() / autoArrange()          │
│  → 生成可编辑原理图                      │
└─────────────────────────────────────────┘
```

### POC 实施路径

**第一阶段（1~2周）— 工具链验证**
- 搭建立创 EDA 插件开发环境
- 跑通 SDK 基础操作：放置元件、连线、保存
- 手动实现 2~3 个固定模板电路（LDO、MCU 最小系统等）

**第二阶段（2~3周）— AI 接入**
- 搭建后端服务，接入 LLM（GPT-4o / 本地 Qwen）
- 设计 Prompt + JSON Schema，确保输出结构稳定
- 打通 插件 ↔ 后端 ↔ SDK 完整链路

**第三阶段 — 扩展与优化**
- 扩充电路模板库（结合立创开源广场数据）
- 加入 DRC 自动检查反馈循环
- 优化布局美观性算法

## 目标用户

### 主要用户

#### 用户一：小林 — 无硬件背景的软件工程师

**背景：** 后端开发 3 年，想做一个自己的 IoT 小产品（智能插座、传感器采集器等）。软件层完全能搞定，但一到硬件就卡住了。

**当前痛点：**
- 去 GitHub / 立创开源广场找参考，要对比 5~10 个项目才能找到接近需求的
- 看懂别人的原理图耗时大，还不确定能否直接套用
- 不知道该选哪个芯片，选错了又要重新研究
- 大多数情况下选择放弃，或花钱找人代画

**成功样子：** 能像描述软件需求一样描述电路需求，工具自动生成原理图，审一遍就能去打板。

---

#### 用户二：阿杰 — 独立创客 / Maker

**背景：** 有动手热情，玩过 Arduino / ESP32 开发板，会焊接，想把 idea 做成真正的产品而不只是开发板原型。

**当前痛点：**
- 开发板转 PCB 这一步完全不知道怎么下手
- 抄电路抄得不踏实——不清楚某个器件的作用，不敢改
- 从想法到打板周期太长，热情消耗在等待和纠结中

**成功样子：** 把脑子里的产品方案快速变成可打板原理图，迭代速度加快，真正把 idea 落地。

**使用场景：** 快速生成参考原理图作为起点，在其基础上修改，提升效率，而非替代专业判断。

## 成功指标

### POC 验收标准（最优先）

> **核心验收条件：** 通过自然语言对话，插件能在立创 EDA 中生成包含真实可用元件的原理图。

| 验收项 | 标准 |
|--------|------|
| **对话驱动** | 用户用自然语言描述需求（如"ESP32最小系统"），无需了解元件型号 |
| **元件真实性** | 生成的所有元件均来自立创 EDA 官方元件库，含正确封装和引脚定义 |
| **可采购性** | 关键元件可在立创商城搜索到，有实际库存（避免使用停产/稀缺器件） |
| **原理图可用** | 生成的原理图能通过立创 EDA 基础 DRC 检查，无严重错误 |
| **端到端跑通** | 从输入文字 → 生成原理图全流程无需手动干预即可完成 |

---

### 用户成功指标（产品阶段）

**直接成功：** 用户生成了一张"可直接或微调后打板"的原理图
- 元件全部在立创商城有货
- 连接关系符合数据手册推荐电路
- 用户无需从零学习 EDA 操作

**北极星指标（打板转化率）：**
- 用户从生成原理图 → 实际下单嘉立创打板的比例
- 这是"人人都是创客"愿景的最终体现

---

### 质量红线（非功能要求）

- ❌ 生成的元件在立创商城无法找到 → 不合格
- ❌ 引脚连接明显错误（电源接地短路等）→ 不合格
- ❌ 使用不通用/小众元件导致采购困难 → 不合格
- ✅ 优先使用立创商城"基础库"或"扩展库"中的主流通用元件

## MVP 范围

### POC 目标电路

**首个跑通目标：LDO 电源模块**
- 选择理由：电路结构简单稳定（3~5 个元件）、应用极广、新手最常用的第一个模块
- 典型形态：输入电压（如 5V）→ AMS1117-3.3 LDO → 输出 3.3V，搭配输入/输出滤波电容
- 验证价值：跑通这一个即可证明"对话 → 原理图"完整链路可行

---

### MVP 核心功能（必须有）

| 功能 | 描述 |
|------|------|
| **自然语言输入** | 用户在插件面板输入电路需求描述 |
| **AI 意图解析** | LLM 解析用户需求，映射到对应电路模板 |
| **原理图自动生成** | 调用立创 EDA SDK，在画布上放置元件、连线 |
| **立创元件库对应** | 生成的元件来自立创 EDA 官方库，引脚/封装正确 |

---

### 明确不在 MVP 范围内

| 功能 | 说明 | 规划阶段 |
|------|------|----------|
| AI 自动查询立创商城实时元件 | POC 阶段使用预置元件映射表 | V2 |
| 多轮对话修改原理图 | 每次生成为独立会话 | V2 |
| PCB Layout 自动生成 | 超出范围 | 未定 |
| 多 EDA 工具支持（KiCad 等） | 专注立创 EDA | 未定 |
| 复杂电路（DDR、RF、高速信号）| 风险高，不适合 POC | 未定 |
| DRC 自动修复 | 只提示不修复 | V2 |

---

### 未来愿景（V2+）

- **多轮对话修改**：支持"帮我把输出换成 1.8V"、"加一个使能引脚"等增量修改
- **立创商城实时选型**：AI 查询商城 API，优先推荐有货、价格合理的元件
- **电路模板库扩展**：MCU 最小系统、USB 供电、电机驱动、传感器接入等常用模块
- **生成质量评估**：AI 给出设计建议和风险提示
- **社区模板共享**：用户可上传/下载 AI 生成过的优质电路模板
